Fraction finder

Introduction

This tutorial shows a simple way to reconstruct a fraction form \(\frac{a}{b}\) from a discrete rational value (e.g. \(0.142857142...\) ). It follows an intuitive, but sometimes slow algorithm.

Algorithm

The algorithm uses two variables \(a\) and \(b\) as numerator and denominator of the fraction which are altered during the process. In the beginning they are both set to \(1\) giving the fraction \(\frac{a}{b}\) a start value of \(1\). If the discrete value is less than the fraction, value \(b\) is increased which leads to a decreasement of the whole fraction. Else, \(a\) is increased, increasing the whole fraction. The process is repeated untill no difference is detected between the fraction and the discrete value. If the value was rational, the algorithm found the reduced fraction which produces the digital representation. If the value was intended to be irrational (e.g. \(\sqrt{2}\) ) the algorithm returns the first fraction that the machine can not tell apart from the real value.

Implementation

def discrete_to_fraction(value):
    a = 1
    b = 1
    while a / b != value:
        if a / b > value:
            b += 1
        else:
            a += 1
        return a, b

Conclusion

The given algorithm is a simple and precise way to determine the numerator and denominator of a given discrete fraction. The weakness of the algorithm is that it takes it \(m+n\) tries to guess a fully reduced fraction \(\frac{m}{n}\).
